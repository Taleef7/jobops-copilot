# Azure Deployment

## Phase 1 Azure Targets

- Azure Static Web Apps for the frontend
- Azure Functions for API endpoints and AI orchestration
- Azure Blob Storage for resumes, snapshots, and generated reports

## Phase 2 Azure Targets

- Azure Database for PostgreSQL for durable CRM storage
- Azure Application Insights for monitoring and tracing
- Azure Key Vault for secret management

## Optional Phase 3 Additions

- Azure AI Search for semantic retrieval across job and resume history
- Azure OpenAI if the account is available
- Azure Container Apps if self-hosting n8n becomes useful

## Deployment Notes

- Keep environment variables out of source control.
- Use `.env.example` locally and Azure application settings in hosted environments.
- Prefer structured JSON logs for API and workflow observability.

## Azure PostgreSQL Bootstrap

For local development against Azure Database for PostgreSQL Flexible Server:

1. Create the server in an allowed subscription region and keep the cheap dev configuration:
   - Burstable compute
   - `Standard_B1ms`
   - `32 GiB` storage
   - high availability disabled
2. Set `DATABASE_URL` in `apps/api/.env` using the Azure endpoint and `sslmode=require`.
3. Run the bootstrap script from the repo root:

```bash
npm run db:init --workspace @jobops/api
```

4. Start the API with `npm run dev:api`.
5. Verify `GET /api/health` returns `mode: "postgres"`.

Notes:

- The schema is Azure-compatible without `pgcrypto`; UUIDs are generated by the API and seed scripts.
- The bootstrap script is idempotent, so rerunning it is safe after a schema reset.
